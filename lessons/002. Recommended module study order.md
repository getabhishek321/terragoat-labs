RECOMMENDED ORDER OF MODULE STUDY (SAFEST -> MOST CRITICAL)
1) S3

– public buckets, no encryption, bad bucket policies

2) IAM

– wildcards, admin permissions, bad trust policies

3) EC2

– SG 0.0.0.0/0, IMDSv1, hard-coded keys

4) Security Groups / Networking
5) Secrets Manager / Lambda leaks
6) API Gateway

– unauthenticated public endpoints

7) KMS

– weak key policies

8) RDS

– no encryption, public access

9) DynamoDB

– no PITR, no encryption, wide IAM access
